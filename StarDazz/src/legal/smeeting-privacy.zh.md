# smeeting 隐私政策

**最近更新：** 2026 年 4 月 23 日

**StarDazz**（“我们”）发布 **smeeting** Android 应用（“本应用”）。本政策说明本应用如何在您的设备上处理敏感数据，并与 [GitHub 上公开 README](https://github.com/YihanLi-erisaer/smeeting/blob/master/README.md) 中描述的技术行为一致，便于您理解数据流向并满足应用商店对用户数据透明度的常见要求。

---

## 1. 适用人群（含儿童）

本应用**并非**以 **13 周岁以下儿童**为主要受众进行设计或推广。若您未满 13 周岁，请在**家长或监护人**安装、监督使用并同意本政策的前提下再使用 smeeting。

若您认为儿童在不当情形下使用了本应用，请通过第 11 节方式联系我们。我们不会故意为营销目的收集儿童的个人信息。

---

## 2. 数据处理概要（与产品设计一致）

- **流式语音识别（ASR）：** 麦克风音频在**设备本地**通过原生管线（Sherpa-NCNN / ncnn）处理。项目文档说明流式识别**不使用云端 ASR**。
- **转写与历史记录：** 会议文本及相关条目**保存在本地**（例如 README 架构中描述的 Room / 本地历史）。
- **可选摘要：** 您可通过 HTTPS **下载**量化 **Qwen3** ncnn 权重包；下载完成后，摘要针对您已有的转写文本在**设备本地**运行。
- **网络：** 网络主要用于**您主动发起**的可选模型资源下载（参见 README 中列出的下载镜像，例如 ncnn 模型相关镜像）。流式 ASR 本身不会将您的麦克风音频发送到 StarDazz 运营的服务器进行识别。

---

## 3. 处理的信息类别

| 类别 | 说明 |
|------|------|
| **音频** | 用于实时 ASR 的短时麦克风输入；在设备上处理。 |
| **文本 / 转写** | 本应用生成或保存的文本；保存在本地，除非您自行导出或分享。 |
| **可选模型文件** | 为可选 LLM 摘要下载的较大资源包；保存在源码树中说明的应用内部目录。 |
| **设备与系统数据** | Android 系统可能独立收集诊断信息；请参阅设备厂商与 Google 账户相关设置。 |

我们**不会出售**您的个人信息。

---

## 4. Android 权限

本应用可能申请的权限包括但不限于：

- **`RECORD_AUDIO`** — 用于采集音频以进行设备端 ASR。
- **`INTERNET`** — 用于按仓库文档**下载**可选模型包（推理仍在设备端）。

具体列表以您安装的版本及系统权限弹窗为准。

---

## 5. 存储、保留与删除

数据保存在设备上，存储方式与开源仓库说明一致（含可选 Qwen3 资源的内部目录）。您可在应用提供的界面中删除内容。**卸载**应用将按 Android 规则清除应用沙盒数据。

---

## 6. 第三方与开源组件

smeeting 依赖 **开源** 原生组件（如 Sherpa-NCNN、ncnn、可选 Qwen3 ncnn 包）。当您下载模型时，设备会访问项目文档中列出的第三方 **HTTPS** 端点；与任何文件下载类似，对方可能记录常规服务器日志（如 IP、请求时间）。

源代码托管于 **GitHub**，采用 **Apache License 2.0**；使用 GitHub 受 GitHub 自身政策约束。

---

## 7. 安全

我们与 README 一致，强调麦克风数据、ASR 与可选 LLM 推理以**设备端**为主。任何存储与处理方式都无法保证绝对安全——请保持系统更新并妥善保管设备物理访问。

---

## 8. 跨境与国际用户

若您在境外使用本应用，处理仍主要发生在您的设备上。下载流量可能由您的运营商在全球范围内路由。

---

当功能或法规要求变化时，我们可能更新本政策，并更新文首**最近更新**日期；重大变更时我们会在本站和/或 GitHub 仓库中另行说明。
